# План Ручного Тестирования

## Среда

- Движок: `Godot 4.6.2 Standard`
- Точка входа: `res://scenes/Briefing.tscn`
- Основная цель проверки: пройти весь маршрут `Briefing -> Main -> SecondaryGenerator -> Evacuation -> Debrief`

## Базовая Проверка

- Проект стартует со сцены титульного экрана `Briefing`.
- Кнопка старта переводит в обучение `ControlsLab`.
- В обучении работает динамическая подсказка: при использовании клавиатуры/мыши показываются кнопки ПК, при вводе с геймпада - настройки геймпада (чувствительность, мертвая зона).
- Управление работает: `WASD` / Левый стик, мышь / Правый стик, `E` / `A`, `Shift` / `L3`, `Esc` / `Start`.
- Взаимодействие с тренировочным терминалом выдает успешный feedback.
- Взаимодействие с дверью выхода запускает `Main`.
- При движении по полу слышны concrete footstep-звуки.
- Все ключевые тексты интерфейса отображаются на русском.
- Кнопки в брифинге, HUD-решениях и дебрифе используют текстурированный UI-скин.

## Проверка Сцены `Main`

- В цехе есть новые текстуры пола/стен (Kenney Prototype), потолок, балки, освещение, разметка, пропсы и рабочий выход, а сцена не выглядит пустой коробкой.
- У компрессора видны дым, тревожная лампа и аварийная кнопка.
- Подсказки меняются динамически (без хардкода префикса `[E]`), в зависимости от последнего устройства ввода.
- После нажатия аварийной остановки обновляются `objective`, `hint`, `banner` и `feedback`.
- После входа в safe zone появляется панель решения.
- Выбор `Подать сигнал / вызвать помощь` открывает задачу выхода из цеха.
- После взаимодействия с выходом происходит переход в `SecondaryGenerator`.

## Ошибки В `Main`

- Ничего не делать около `24s`.
  Ожидаемо: переход в `Debrief` с провалом.
- Взаимодействовать с панелью спасения оборудования.
  Ожидаемо: переход в `Debrief` с провалом.
- В safe zone выбрать `Пытаться спасать оборудование`.
  Ожидаемо: переход в `Debrief` с провалом.

## Проверка Сцены `SecondaryGenerator`

- После успешного этапа в `Main` загружается отдельная сцена резервного генератора.
- На сцене виден `GEN-02`, панель отключения и шлюз эвакуации.
- Пока генератор не отключён, попытка уйти в шлюз блокируется warning-сообщением.
- После нажатия на панель `GEN-02` меняются цвет панели, статусный свет и указания в HUD.
- После отключения генератора взаимодействие со шлюзом переводит в `PowerShutdown`.

## Проверка Сцены `PowerShutdown`

- Загружается комната с 3 щитками на разных стенах.
- При попытке покинуть помещение до отключения щитков выводится предупреждение.
- При отключении каждого щитка он меняет цвет и отображается прогресс (1/3, 2/3).
- При отключении всех трёх щитков включается зелёный свет над дверью, разрешая выход.
- При выходе происходит переход к `Evacuation`.

## Проверка Сцены `Evacuation`

- Сцена не содержит лишнего пропа по центру.
- Видны маршрут к пункту сбора, fenced-зона, canopy/industrial props и понятная зелёная разметка.
- На пункте сбора доступно взаимодействие `Подтвердить эвакуацию`.
- После подтверждения загружается `Debrief` с успешным результатом.
- В списке итогов есть пункт про отключение резервного генератора и щитков.

## Ошибка На Этапе Эвакуации

- Во время `Evacuation` взаимодействовать с панелью возврата в опасную зону.
  Ожидаемо: переход в `Debrief` с неуспешным результатом. При этом к `GameSession` добавляется ошибка.

## Проверка `Debrief`

- На экране итогов видны заголовок, описание результата и список ключевых выводов.
- Между результатами и списком выводов отображается строка `Статистика сессии: Время - MM:SS | Ошибок - N`.
- Кнопка `Повторить тренировку` возвращает в `Main` и не сбрасывает общее время и ошибки сессии.
- Кнопка `Вернуться на титульный` возвращает в `Briefing` и после нового старта сбросит время и ошибки.
